# PRD Template for B2B Management Dashboards

Use this structure when writing a PRD for a new management/operation dashboard.
Fill in each section with the user's specific business context.

## Problem Statement

Describe the current manual process or pain point. Explain:
- What is the current workflow (people, tools, manual steps involved)?
- What problems arise from the manual/current approach?
- Quantify the impact if possible (time wasted, errors, missed opportunities)
- Why is this problem worth solving now?

## Solution

Describe the proposed system. Explain:
- What is the core replacement/improvement over the current approach?
- What are the key capabilities the system will provide?
- What is the scope boundary (what's in, what's out for v1)?
- What is the expected outcome (efficiency gain, error reduction, etc.)?

## User Stories

Group stories by feature area. Each story follows the format:
`As [user role], I want [action], so that [benefit].`

Number stories sequentially across all sections for easy reference.
